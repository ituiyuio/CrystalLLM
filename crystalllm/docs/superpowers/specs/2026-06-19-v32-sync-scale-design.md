# v32 设计: 同步扩规模 (扩散 drafter + v28.5 → v32)

> **Q: 把 v31 的扩散 drafter + AR verifier 框架同步扩大 (drafter 28M→100M, verifier 28L→32L, 数据 19K→50K), 能达到什么水平?**
> **A: 目标 206ms → ~150ms, PPL 2.39 → < 2.0, 接受率 95.5% → 97%+**

## 1. 设计哲学

### 1.1 v31 成功后, 同步扩大规模

v31 证明了"扩散 + AR 融合"的可行性:
- 扩散 drafter + AR verifier SpS = 206ms (2.73x 加速)
- 接受率 95.5% (远高于 v26 AR drafter 65%)

**下一步**: 在这个框架上同步扩大规模, 进一步提升性能.

### 1.2 为什么"同步扩大"

不只扩一个维度, 因为它们互相影响:
- **drafter 越大** → 越接近 verifier 分布 → 接受率越高
- **verifier 越大** → PPL 越低 → 质量越好
- **数据越大** → 训练更充分 → 所有模型更好

**同步扩大** = 三者都受益, 复合效应.

## 2. 三个维度的扩大

### 2.1 扩散 Drafter: 28M → 100M

| 配置 | v31 | **v32** |
|---|---|---|
| ResBlock 数 | 6 | **12** |
| Hidden dim | 1024 | **1536** |
| 参数量 | 28M | **100M (3.6x)** |
| K (output tokens) | 8 | **8 (不变)** |
| 训练步数 | 4000 | **6000** |

**预期**: 接受率 95.5% → 97%+ (drafter 拟合 verifier 分布更准).

### 2.2 AR Verifier: 28L → 32L

| 配置 | v28.5 (SOTA) | **v32** |
|---|---|---|
| Layers | 28 | **32** |
| embd | 1280 | **1600** |
| head | 20 | **25** |
| 参数量 | 555M | **~1.2B (2.2x)** |
| Warm-start | from v23 | **from v28.5** |
| 训练数据 | 19K | **19K (复用)** |

**预期**: PPL 2.39 → < 2.0 (-15%+).

### 2.3 训练数据: 19K → 50K

| 数据 | v31 | **v32** |
|---|---|---|
| 来源 | v24 train | **v24 + v23 高质量子集** |
| 样本数 | 19,307 | **50,000** |
| 字符量 | 77.9M | **~200M** |

**预期**: encoder + prior + decoder 都有更好数据.

## 3. 实施路线

### Phase 1: 重训 v32 Verifier (~20 min)

**train_v32_decoder.py**:
- 32L × 1600 × 25 (~1.2B)
- Warm-start from v28.5 (复用前 28 层)
- 新 4 层随机初始化
- 数据: v24 train (19K) - 不变, 避免 v28 数据分布问题
- 4000 步, LR=2e-5 (微调)
- 训练时间: ~20 min (1.2B 模型)

**评估**:
- PPL < 2.0 (目标)
- 速度 ~640ms 单 forward

### Phase 2: 数据扩展 (~10 min)

**build_v32_data.py**:
- 从 v24 train (19K) + v23 train (6K) = 25K
- 或者从 raw_v23 子集 50K
- 重新编码 (encoder + diffusion prior)
- 输出: cached_v32_z.npz

### Phase 3: 重训扩散 Drafter (~20 min)

**train_v32_diff_drafter.py**:
- 100M, 12 ResBlock × 1536
- 数据: 收集 v32 verifier 的输出 (50K 样本)
- 训练目标: v32 verifier 输出
- 6000 步

### Phase 4: SpS 集成 (~10 min)

**eval_v32_sps.py**:
- v32 verifier (1.2B) + v32 drafter (100M)
- SpS K=8 (或调大)
- 评估: 速度, PPL, 接受率

## 4. 预期结果

| 指标 | v31 | **v32 (估)** | 变化 |
|---|---:|---:|---|
| **速度** | 206ms | **150-180ms** | -15% |
| **接受率** | 95.5% | **97%+** | +1.5% |
| **PPL** | 2.39 | **< 2.0** | -15%+ |
| 模型总计 | 583M | **~1.3B** | 2.2x |
| 数据量 | 19K | **50K** | 2.6x |

**核心加速来源**:
- Drafter 越大 → 接受率越高 → rounds 越少
- Verifier 越大 → PPL 越低 → 质量越好
- 数据越大 → 训练越充分 → 所有都好

## 5. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 1.2B 模型训练慢 | 中 | 中 | warm-start from v28.5, 只训新 4 层 |
| 1.2B 模型 OOM | 低 | 高 | 用 float16 / gradient checkpointing |
| 50K 数据混合分布 | 中 | 中 | 子采样避免短文 (与 v28 一致教训) |
| 100M drafter 太慢 | 低 | 中 | 仍 1 forward / round, 可接受 |
| 总速度没比 v31 快 | 中 | 高 | 接受率提升是关键 |

## 6. 与 v28 的对比 (避免重复失败)

v28 失败原因:
1. 数据分布不匹配 (ext_v23 短文 vs val 长文)
2. 训练步数不够 (2000 < 4000)
3. 无 warm-start

**v32 修复**:
1. 数据用 v24 19K (不变) + v23 高质量子集, 避免 ext_v23
2. 训练步数 4000 (不变)
3. **Warm-start from v28.5** (关键)

## 8. 下一步 (v33+)

### 8.1 短期 (v33): 更大扩散 drafter (100M → 200M)

- 进一步提升接受率
- 减少 rounds 数

### 8.2 中期 (v34): 完全扩散 + AR

## 9. 总结

**v32 = v31 框架 + 同步扩规模**:
- 扩散 drafter: 28M → 100M (3.6x)
- AR verifier: 28L → 32L (2.2x 参数)
- 训练数据: 19K → 50K (2.6x)
- 目标: 206ms → ~150ms, PPL 2.39 → < 2.0

**核心思想**: 在 v31 成功的框架上, 同步扩大三个维度, 复合提升性能.
