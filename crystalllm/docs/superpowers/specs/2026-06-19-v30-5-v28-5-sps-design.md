# v30.5 设计: v28.5 + 投机解码 (Speculative Decoding)

> **Q: 用 v28.5 (28L, 555M) 作 verifier + 重训 90M drafter 适配 v28.5 + SpS K=5, 能加速多少?**
> **A: 目标 828ms → ~600ms (1.4x 加速). PPL 维持 2.39 (verifier 决定).**

## 1. 背景

### 1.1 v30 失败教训

v30 试图重做 v27 扩散 KV 路线, 即使扩大 9 倍, 仍失败:
- 速度 1109ms (vs v28.5 828ms, **慢 34%**)
- cos_sim ~0.5 (扩散 KV 拟合失败)
- 生成质量乱码

**根因**: KV cache 在 ~500M 模型上 launch overhead 主导, KV cache 节省的 compute 完全被 launch overhead 淹没.

### 1.2 v30.5 思路转变

**抛弃 KV cache 优化**, 改为**减少 AR forward 次数**:

```
v30.5 = v28.5 + SpS (Speculative Decoding)

每 round:
1. drafter 草拟 K=5 tokens (5 × 1.5ms = 7.5ms, 小模型)
2. verifier 1 forward 验证 K tokens (1 × 7.67ms)
3. 接受匹配前缀

接受率 ~65% (drafter PPL ~4 vs verifier PPL 2.39)
平均接受 3.27 tokens/round
需 100 / 3.27 ≈ 31 rounds
总时间 ≈ 31 × (7.5 + 7.67) = 470ms (vs 828ms AR)
```

## 2. 关键问题

### 2.1 v26 drafter 能不能复用?

v26 drafter 是 **12L × 768 × 12** (90M), 训练时模仿 v25 (24L × 1280 × 20).
v28.5 是 **28L × 1280 × 20** (555M).

**架构差异**:
- 层数: 12 vs 28 (drafter 必须更深才能模仿 v28.5? 不, drafter 可以比 verifier 小)
- 维度: 768 vs 1280 (不能直接复用)

**结论**: **必须重训 drafter**.

### 2.2 drafter 规模选择

| drafter | 层数 × embd × head | 参数 | 训练时间 | 接受率估计 |
|---|---|---:|---:|---:|
| 90M (12L × 768 × 12) | 小 | 90M | 8 min | ~65% |
| 200M (16L × 1024 × 16) | 中 | 200M | 15 min | ~75% |
| 300M (20L × 1024 × 16) | 大 | 300M | 20 min | ~80% |

**选 90M**: 与 v26 一致, 训练快, 接受率 ~65%.

如果 65% 接受率不够, 再升级到 200M.

## 3. 架构

### 3.1 drafter (重训)

```
Drafter:
- 12L × 768 × 12 (~90M)
- T=512, D_Z=256
- 与 v26 架构一致
- 训练数据: v24 train (19K)
- 训练目标: 模仿 v28.5 的输出
- 训练步数: 4000 (与 v26 一致)
```

### 3.2 投机解码流程

```python
def gen_sps(n_ar=100, k=5):
    z = sample_prior(1, n_steps=5)  # 50ms 一次性
    cur = [BOS]

    while len(cur) - 1 < n_ar:
        # Stage 1: drafter 草拟 K tokens
        draft = list(cur)
        for _ in range(k):
            logits = drafter(z, draft[-T:])
            draft.append(argmax(logits[-1]))

        # Stage 2: verifier 1 forward 验证 K tokens
        v_logits = verifier(z, draft[-T:])
        v_tokens = [argmax(v_logits[i]) for i in range(k)]

        # Stage 3: 接受匹配前缀
        n_acc = 0
        for j in range(k):
            if draft[1+j] == v_tokens[j]:
                n_acc += 1
            else:
                break

        cur.extend(draft[1:1+n_acc])
        if n_acc < k:
            cur.append(v_tokens[n_acc])

    return cur
```
