# 设计文档：CrystaLLM 混合生成系统

## 1. 概览
CrystaLLM 将文本生成分解为两个阶段：
- **阶段 I（扩散域定位）**：在连续潜变量空间运行轻量扩散过程，快速将随机噪声凝结为承载全局语义意图的潜向量 **z**。
- **阶段 II（自回归寻路）**：以 **z** 为全局条件，通过因果注意力机制逐 token 生成具体文本。

这种分解使全局规划（语义布局）与局部实现（token 选择）解耦，获得扩散的并行效率和自回归的局部精度。

## 2. 系统架构
```
输入上下文 (可选)
        │
        ▼
┌─────────────────────────┐
│  潜变量扩散模块          │
│  (Denoising MLP/Transformer) │
│   z_T ~ N(0, I)         │
│   z_T → z_{T-1} → ... → z_0 │
│   z = z_0               │
└────────────┬────────────┘
             │ z (语义域坐标)
             ▼
┌─────────────────────────┐
│  自回归解码器            │
│  p(x_t | x_{<t}, z)     │
│  逐 token 采样           │
│  x_1, x_2, ..., x_N     │
└────────────┬────────────┘
             │
             ▼
         完整文本
```

### 2.1 潜变量扩散模块
- **输入**：可选的上下文文本嵌入（如提示词）拼接到噪声。
- **扩散过程**：采用 DDPM / 流匹配。总扩散步数 T=10~20。在低维潜空间（d=512~2048）运行，计算量极低。
- **网络结构**：小型 Transformer 或 ResMLP，以当前噪声潜变量、扩散时间步、上下文嵌入为输入，预测噪声或速度场。
- **训练目标**：
  \( \mathcal{L}_{\text{diff}} = \mathbb{E}_{t, \epsilon} \left[ \|\epsilon - \epsilon_\theta(z_t, t, c)\|^2 \right] \)，其中 \(c\) 为上下文嵌入。
  或流匹配损失：预测速度场 v。
- **输出**：去噪后的潜变量 z₀，用作解码器的条件。

### 2.2 条件自回归解码器
- **架构**：标准 Transformer 解码器（因果注意力）。
- **条件注入方式**：
  - 在输入 token 嵌入序列的开头添加可学习 [Z] token，其嵌入由潜变量 z 经线性投影得到。
  - 或通过交叉注意力，让解码器每一层都 attend 到 z。
- **生成**：使用 top‑p / top‑k 采样，温度可调。
- **训练目标**：标准交叉熵损失 \( \mathcal{L}_{\text{ar}} = -\sum \log p_\theta(x_t | x_{<t}, z) \)，其中 z 来自训练样本的编码（预训练时）或扩散模块的输出（联合训练时）。

### 2.3 两阶段调度与联合训练
- **训练阶段**：
  - 方案 A（冻结编码器）：用预训练的自编码器将文本编码为潜变量 z（如 Sentence‑BERT 或 VAE），仅训练扩散模块；然后冻结扩散模块训练自回归解码器。
  - 方案 B（端到端微调）：先训练 VAE 或直接使用离散潜变量（如 VQ‑VAE），然后联合微调扩散+解码器。
- **推理调度**：
  1. 运行扩散模块 K=10 步得到 z。
  2. 将 z 送入自回归解码器，逐 token 生成完整序列。
  3. 可选：对解码器进行少量步的"反思式精修"（如使用投机采样加速）。

## 3. 数据与预处理
- **训练语料**：C4、OpenWebText、维基百科、书籍等。
- **预处理**：分词器使用 SentencePiece（如 Llama 2 tokenizer）。
- **潜变量训练**：使用句子/段落级别的编码器（如 DPR、SimCSE）提取连续语义向量作为 z 的真值，训练扩散模块学习其分布。

## 4. 技术决策
- **扩散范式**：采用 **流匹配**（Flow Matching）替代 DDPM，因为它能以更少步数（5‑10 步）生成高质量样本。
- **潜变量维度**：d=1024，平衡表达力与扩散效率。
- **解码器规模**：设计 500M–1.5B 参数（与 GPT‑2 XL/Neo 相当）。
- **注意力机制**：解码器使用闪存注意力（Flash Attention）以节省内存。
- **推理加速**：自回归阶段可以使用 KV 缓存，且扩散阶段与解码器可异步流水线化。

## 5. 评估策略
### 5.1 自动指标
- **生成质量**：困惑度（PPL）在测试集上评测。
- **效率**：平均生成延迟（秒/词）、吞吐量（tokens/sec）。
- **语义控制**：给定不同的 z（通过插值、属性向量），测量情感分类准确率、主题一致性（NPMI）。

### 5.2 人工评估
- 对比实验：纯自回归 vs. 纯扩散 vs. CrystaLLM。
- 标准：流畅度、信息性、创造力、与意图的匹配度。

## 6. 实验计划
- **消融研究**：
  - 扩散步数（T=1, 5, 10, 20）的影响。
  - 潜变量注入方式（前缀 vs. 交叉注意力）。
  - 有无上下文条件对比。
- **基线模型**：
  - GPT‑2 同规模的自回归模型。
  - 纯掩码扩散语言模型（MDLM）。
  - 使用 VAE 条件化的自回归模型。
- **扩展试验**：在代码生成、故事创作等具体任务上微调并评估。

## 7. 风险与缓解
| 风险 | 可能性 | 缓解措施 |
|------|--------|----------|
| 潜变量表达能力不足，解码器忽视 z | 中 | 增加潜变量维度，使用对抗训练或互信息最大化。 |
| 两阶段训练难以收敛 | 中 | 采用分阶段训练，先训编码器，再训扩散，最后联合微调。 |
| 推理速度未达预期 | 低 | 使用流匹配减少步数；解码器可使用投机解码加速。 |
| 生成文本带有扩散随机性导致的错误 | 低 | 在后处理阶段引入微小自回归精修或重排序。 |
