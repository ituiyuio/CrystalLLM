# §6 CWF Phase 1: The Decisive Experiment

If the closure property is necessary for continuous representation, a purpose-built architecture (CWF) should outperform patchwork. We designed and tested CWF on the Lorenz chaotic system as a stress test for the closure hypothesis. The result: \textbf{TOTAL\_FAIL}.

## 6.1 Why Lorenz?

The Lorenz system
\begin{align*}
    \dot{x} &= \sigma(y - x) \\
    \dot{y} &= x(\rho - z) - y \\
    \dot{z} &= xy - \beta z
\end{align*}
(with standard parameters $\sigma = 10$, $\rho = 28$, $\beta = 8/3$) is the canonical example of deterministic chaos. Its maximum Lyapunov exponent $\lambda \approx 0.906$ means that small errors are amplified by a factor of $e^{\lambda t}$ over time $t$.

For a model to "learn the Lorenz system", it must either:
\begin{enumerate}
    \item Match the true ODE dynamics (with sufficient accuracy to stay below the Lyapunov error amplification threshold), or
    \item Learn an internal representation that has the same Lyapunov structure (without explicitly representing the ODE).
\end{enumerate}

Either way, the model must learn a \emph{continuous dynamics} on a \emph{continuous state space}. This is the prototypical task that autoregressive token prediction was not designed for, and the natural test for the closure hypothesis.

The Oracle baseline (RK4 integration of the true Lorenz ODE) provides the physics upper bound: its effective prediction time (EPT) at correlation threshold 0.9 is 100 steps.

## 6.2 The Three Models

We compared three models on the same training and evaluation data:

\subsection{CWF (Closed Waveformer)}

A purpose-built continuous architecture with six closed components:
\begin{itemize}
    \item FFT-based input encoding (continuous $\to \mathbb{D}^d$)
    \item Lie group position rotation ($\mathrm{SO}(n)$-isometry, preserves norm)
    \item Complex-valued attention (unit-phase weighting, no softmax)
    \item Complex Siren FFN (continuous activation on $\mathbb{C}^d$)
    \item Born-stable normalization ($\|\psi\| \leq 1 - \epsilon$ projection)
    \item Born rule decoder (continuous $\to$ continuous, only at final layer)
\end{itemize}

Architecture details: 3-channel input (one per Lorenz dimension), FFT encoder per channel, concatenated complex state in $\mathbb{D}^{96}$ ($d = 32$ per channel, $3 \times 32 = 96$), one CWF block, Born rule decoder per channel.

Parameters: $0.98$M. Closure verified across 10 random batches at initialization.

\subsection{AR+VQ Baseline}

A standard autoregressive Transformer with a VQ-VAE bottleneck on the input:
\begin{itemize}
    \item VQ encoder: continuous Lorenz state $\to$ discrete token IDs (codebook size 512)
    \item Token embedding $\to$ Transformer encoder $\to$ output head $\to$ next-token logits
    \item Output head also projects back to continuous via softmax-weighted codebook expectation
\end{itemize}

The VQ bottleneck simulates the "language model on continuous physics" use case: discrete tokens forced to represent a continuous signal.

Parameters: $0.76$M (including VQ codebook of $512 \times 3 = 1536$).

\subsection{Oracle (RK4)}

Direct Runge-Kutta integration of the true Lorenz ODE. No parameters, no training. Provides the physics upper bound.

\section{6.3 Training Protocol}

All models trained for 500 steps on 200 trajectories (1024 steps each) generated by RK4 from random initial conditions in the attractor basin. Sequence length 256. AdamW, LR $1 \times 10^{-3}$, cosine schedule. Trained on 1-step teacher-forced prediction (next-state given previous 256 states).

\section{6.5 Diagnostic: Why Did CWF Fail on Rollout?

The rollout failure, despite one-step prediction success, indicates that CWF's forward pass is not learning the \emph{dynamics} of the Lorenz system. It is learning a \emph{local next-state mapping}, similar in nature to the autoregressive baseline.

\subsection{Architectural absence: ODE evolution}

CWF's forward pass is:
\begin{align*}
    \psi^{(0)} &= \text{FFT encoder}(x_{1:S}) \\
    \psi^{(1)} &= \text{CWF block}(\psi^{(0)}) \\
    \hat{y} &= \text{Born decoder}(\psi^{(1)})
\end{align*}

This is a \textbf{single-shot mapping from a sequence to a prediction}. The architecture lacks:
\begin{itemize}
    \item \textbf{Time-step embedding}: the model does not know the integration step $\Delta t$
    \item \textbf{Recurrent state}: there is no $h_{t+1} = f(h_t, x_t)$ propagation
    \item \textbf{ODE solver}: there is no numerical integration of learned dynamics
\end{itemize}

Without these components, CWF cannot learn the \emph{continuous evolution} required for chaotic dynamics. It learns a single-step prediction rule that works locally but fails globally.

\subsection{Comparison to Oracle}

The Oracle is:
\begin{align*}
    \hat{y}_{t+K} = \int_t^{t+K} f_\text{Lorenz}(s) ds
\end{align*}

This is a true continuous evolution. The Oracle knows the dynamics explicitly; it integrates the ODE step by step.

CWF attempts to approximate this integration implicitly through a single-shot forward pass. The information bottleneck of a single-shot mapping (input is a 256-step sequence, output is a single 3D state) is fundamentally too restrictive to encode the continuous dynamics.

\section{6.6 The Verdict}

CWF achieved \textbf{partial success}: closure was verified, one-step prediction worked, and the architecture was implemented correctly. But CWF failed the GO/NO-GO gate: EPT@0.9 was only 2 steps, far from the Oracle's 100 steps.

The failure was \textbf{not} in the closure property itself---closure was maintained throughout training. The failure was in the absence of ODE evolution machinery. CWF is a "sequence mapper" that learns to predict the next state from a context window; it is not a "dynamics learner" that models the continuous evolution.

By Theorem 5.1, closure is necessary for continuous representation. But closure is not sufficient: a closed architecture that maps sequence $\to$ next state cannot model continuous dynamics. The architecture must also include ODE evolution, recurrent state, and time-step embedding.

\section{6.7 What Would Save CWF?

To pass the GO/NO-GO gate, CWF would need:

\begin{enumerate}
    \item \textbf{Neural ODE solver}: replace the single-shot forward pass with $h_{t+1} = h_t + \int_t^{t+1} \text{ContinuousCell}(h_s, x_s) ds$. This is the central missing component.

    \item \textbf{Time-step embedding}: $\Delta t$ must be a model input, so the same architecture can be deployed at any temporal resolution.

    \item \textbf{Recurrent state propagation}: the model must maintain a state across rollout steps, not reset on each call.
\end{enumerate}

These additions are non-trivial engineering (1--2 person-years for a competent researcher), require new GPU kernels for ODE solvers, and re-derive the entire training algorithm (adjoint method for memory-efficient backpropagation).

We did not implement these in our 5-day experimental campaign. We document the failure as an empirical result: closure is necessary but not sufficient.

\section{6.8 What This Proves and What It Does Not}

\textbf{What this proves}:
\begin{itemize}
    \item A closed continuous architecture (verified closure) does not automatically solve continuous dynamics tasks.
    \item Closure is necessary but not sufficient; ODE evolution is also required.
    \item The 1-step prediction advantage of CWF over AR+VQ ($24\times$) does not imply rollout advantage.
\end{itemize}

\textbf{What this does not prove}:
\begin{itemize}
    \item It does not disprove the CWF Manifesto hypothesis that a fully continuous architecture could outperform autoregressive models on language tasks.
    \item It does not show that the 1--2 person-year investment in CWF-with-ODE would fail.
    \item It does not establish autoregressive Transformers as the final answer for continuous data.
\end{itemize}

The CWF Manifesto \citep{wang2026cwf} remains an open research charter. The Phase 1 experiment falsifies the \emph{minimal} CWF prototype (closure without ODE evolution), not the \emph{full} CWF vision (closure + ODE + recurrent state + time-step embedding).
