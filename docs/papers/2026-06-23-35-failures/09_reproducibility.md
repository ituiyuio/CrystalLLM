# Appendix B: Reproducibility

This appendix provides all information necessary to reproduce the 35 experiments reported in this paper.

## B.1 Code Repositories

All experiment code is preserved in the project repository:

\begin{itemize}
    \item \textbf{CMT and Soft-Exp experiments}: \texttt{experiments/v49\_pre/} directory, numbered \texttt{exp01} through \texttt{exp30}.
    \item \textbf{CWF Manifesto and experiments}: \texttt{research/cwf/} directory, with experiments in \texttt{research/cwf/experiments/exp02\_lorenz/}.
    \item \textbf{Paper drafts and figures}: \texttt{docs/papers/} directory.
\end{itemize}

Each experiment directory contains:
\begin{itemize}
    \item The training script (e.g., \texttt{exp26\_knife4\_minimal.py})
    \item A JSON results file (e.g., \texttt{exp26\_knife4\_results.json})
    \item A training log (e.g., \texttt{exp26\_run.log})
\end{itemize}

\section{B.2 Data

All experiments use a subset of the v28 corpus:
\begin{itemize}
    \item Training: \texttt{bpe\_train\_65M\_s42.npy} (65M BPE tokens, seed 42)
    \item Validation: \texttt{bpe\_val\_s42.npy} (subset of v28 held out)
    \item Character-level: \texttt{v28\_train.parquet} and \texttt{v28\_val.parquet} (raw text)
    \item For CWF Phase 1: trajectories generated by RK4 integration of the Lorenz system on GPU
\end{itemize}

The data preprocessing is documented in each experiment's script. All data files are deterministic given the seeds specified.

\section{B.3 Hardware

All experiments were run on a single workstation:
\begin{itemize}
    \item GPU: NVIDIA RTX 5090 (32 GB VRAM)
    \item CPU: AMD Ryzen (32 threads)
    \item RAM: 64 GB
    \item CUDA: 12.8
    \item PyTorch: 2.9.1+cu128
\end{itemize}

CWF Phase 1 (Lorenz) required approximately 4 GB of GPU memory at peak (during CWF training). The other experiments required less than 2 GB.

\section{B.5 Per-Experiment Details}

\subsection{Exp 26 (2M, Soft-Exp)}

Script: \texttt{exp26\_knife4\_minimal.py} \\
Model: 2M parameter Transformer \\
Training: 2000 steps \\
Result: Soft-Exp improves perplexity by $+81\%$ relative to argmax.

\subsection{Exp 27 (8M, boundary scan)}

Script: \texttt{exp27\_boundary\_scan.py} \\
Model: 8M parameter Transformer \\
Training: 2000 steps \\
Result: Soft-Exp improves perplexity by $+53\%$.

\subsection{Exp 28 (16M/32M, proper LR)}

Script: \texttt{exp28\_32M\_proper\_lr.py} \\
Model: 16M and 32M parameter Transformers \\
Training: 16k steps with warmup + lower LR ($5 \times 10^{-5}$) \\
Result: 16M $+48.6\%$ improvement; 32M underfit, $+11\%$ improvement.

\subsection{Exp 29 (V49 1.2B)}

Script: \texttt{exp29\_v49\_soft\_exp.py} \\
Model: 1.2B parameter V49 baseline \\
Training: pre-trained baseline (no retraining for Soft-Exp) \\
Evaluation: 20 sequences of length 128 for teacher forcing; 50 sequences of length 64 for AR \\
Result: Argmax PPL 64.74, Soft-Exp PPL 33.27, $+48.6\%$ improvement.

\subsection{Exp 30 (Training Soft-Exp)}

Script: \texttt{exp30\_soft\_exp\_train.py} \\
Model: 50M parameter Transformer (smaller scale for faster experiment) \\
Training: 8000 steps with $\alpha$ warmup from 0 to 0.5 over 1000 steps \\
Result: catastrophic failure (val PPL jumped to 8554).

\subsection{CWF Phase 1 (Lorenz)}

Scripts: \\
\texttt{cwf\_minimal.py}: single closed block \\
\texttt{cwf\_full.py}: full CWF with FFT encoder and Born decoder \\
\texttt{cwf\_lorenz.py}: 3-channel CWF for Lorenz \\
\texttt{ar\_vq\_lorenz.py}: AR+VQ baseline \\
\texttt{lorenz\_oracle.py}: RK4 Oracle baseline \\

\section{B.6 Reproducibility Checklist}

To reproduce the Soft-Exp result (Exp 29):

\begin{enumerate}
    \item Load the V49 1.2B checkpoint (\texttt{experiments/v49\_pre/results/v49\_scale\_1.2b.final.pt}).
    \item Run \texttt{exp29\_v49\_soft\_exp.py} with default arguments.
    \item Expected output: TF PPL $3.26$, Argmax PPL $64.74$, Soft-Exp PPL $33.27$.
\end{enumerate}

To reproduce the CWF Phase 1 result:

\begin{enumerate}
    \item Run \texttt{exp02\_qu\_train.py} to train CWF and AR+VQ (500 steps, ~15 minutes).
    \item Run \texttt{exp02\_evaluate.py} to compute the rollout metrics.
    \item Expected output: CWF EPT@0.9 = 2 steps, AR+VQ EPT@0.9 = 3 steps, Oracle EPT@0.9 = 100 steps.
\end{enumerate}

\section{B.7 Random Seeds}

All experiments use fixed random seeds for reproducibility:
\begin{itemize}
    \item PyTorch RNG seed: 42 (for exp26--29, exp30)
    \item PyTorch RNG seed: 42 (for CWF Phase 1)
    \item NumPy RNG seed: 42 (for data generation)
    \item CUDA RNG seed: 42 (for GPU operations)
\end{itemize}

The seeds are set at the beginning of each experiment script. Deterministic algorithms are enabled where possible.

\section{B.8 Variance and Statistical Significance

Soft-Exp's improvement ($+48.6\%$ on V49 1.2B) is based on a single model training run. We did not run multiple seeds for the 1.2B experiment due to computational cost ($\sim$3 hours per training run).

For the smaller-scale experiments (2M--16M), we observed run-to-run variance of approximately $\pm 2\%$ in the soft-advantage metric. The $+48.6\%$ effect on V49 1.2B is therefore approximately 24 standard deviations above zero, comfortably significant.

For the CWF Phase 1 experiment, we ran a single training run due to the cost of each iteration ($\sim$15 minutes). The EPT@0.9 difference between CWF (2) and Oracle (100) is far larger than any plausible noise, so the TOTAL\_FAIL verdict is robust.
