# V49 Scale 训练综合报告 — 50M → 200M → 1.2B

**生成日期**: 2026-06-22
**承接**: V49 Formal Training (CMT-Fixed 50M FAIL) + 3-way 诊断 + LM 评估标准 v1.0
**目标**: Scale baseline Transformer 到 1.2B, 验证 V49 路径
**核心结论**: **1.2B val_ppl 2.36 是 best baseline, 但 5/7 PARTIAL (diversity/repetition 仍是 char-level 根本限制). 进一步 scale 不解决. 需要换 tokenization 或数据.**

## 2. 5 维评估对比 (v1.0 标准)

| Model | in_dist PPL | Diversity | Coherent | Repetition | OOD ratio | BPC | **Pass** | Decision |
|---|---|---|---|---|---|---|---|---|
| 50M | 2.7330 | 0.135 | 9/15 | 13/15 | 0.74x | 1.450 | 5/7 | 🟡 PARTIAL |
| 200M | 2.4522 | 0.134 | 10/15 | 10/15 | 0.73x | 1.294 | 5/7 | 🟡 PARTIAL |
| **1.2B** | **2.3564** | **0.157** | **11/15** | **8/15** | **0.73x** | **1.237** | **5/7** | 🟡 PARTIAL |

### 2.1 改善趋势

- ✅ **PPL**: 2.73 → 2.45 → 2.36 (单调下降, 总改善 14%)
- ✅ **Coherent**: 9/15 → 10/15 → 11/15 (单调上升)
- ✅ **Repetition**: 13/15 → 10/15 → 8/15 (单调下降, 总改善 38%)
- ✅ **BPC**: 1.45 → 1.29 → 1.24 (单调下降)
- ⚠️ **Diversity**: 0.135 → 0.134 → 0.157 (1.2B 略好但仍 <0.3)
- ⚠️ **OOD ratio**: 稳定 0.73-0.74x (无改善)

### 2.2 关键观察

**Scale 改善 PPL/Coherent/Repetition,但不解决 Diversity**。所有 3 个规模都是 5/7 PARTIAL,因为 diversity < 0.3 是 char-level code 模型的**结构性限制**:
- char vocab=2261, 200 chars 生成的 char 集合有限
- 1.2B 模型也只用 30+ unique chars,无法达到 0.3 阈值

## 4. 失败根因:为什么 Diversity 不改善

### 4.1 Char-level 根本限制

- Vocab=2261, 字符级预测
- 生成 200 chars 时,即使在"理想"情况下,模型用 top 30 chars 已能生成合理内容
- Diversity 0.3 需要 top 60+ unique chars 在 200 长度内
- 对 char-level code,top 30 chars 已覆盖大部分常用字符

### 4.2 Code 数据的低熵

- v28_train = 27% code (GitHub) + 73% agentic (Claude 对话)
- C++ 代码中: `{}()=;` 等符号字符占 30%+
- 高频字符 (` `, `\n`, `e`, `t`, `n`, `i`, `r`, `o`, `a`, `s`) 占 50%+
- 生成时模型反复用 top chars → low diversity

### 4.3 温度采样与 mode 锁定

- T=0.5 几乎 deterministic,选 argmax = mode 字符
- T=1.0 random,选 top-k 字符
- 即使 T=1.0,也偏向高频字符
- BPE tokenization 可能打破这个限制 (1 token ≈ 4 chars,top 30 tokens = 120+ chars 覆盖率)

## 5. V49 路径最终决策

### 5.1 已验证

✅ **Baseline 路径可行**:
- 1.2B baseline val_ppl 2.36 (vs CMT-Fixed 1.005 但 memorizer, vs Wave 1930 underfitter)
- 真实 LM, 生成连贯代码/英文
- 8-bit AdamW 有效 (peak memory 25GB / 32GB 充足)
- 3h 训练,单 RTX 5090 可行

### 5.2 不通过

❌ **Scale 不解决 diversity**: 50M → 1.2B diversity 0.135 → 0.157 (改善 16% 但仍 <0.3)
❌ **Scale 不解决 repetition**: 13/15 → 8/15 (改善 38% 但仍 >0)
❌ **PPL scale 改善有限**: 2.73 → 2.36 (14% 改善, 边际递减)

### 5.3 V49 最终状态

**V49 = 1.2B baseline Transformer + 8-bit AdamW (V49 canonical)**

```
val_ppl 2.36 | BPC 1.24 | 8/15 repetition | 11/15 coherent
Best PPL among all 4 architectures tested
Real LM, NOT memorizer (unlike CMT-Fixed)
Single RTX 5090, 3.1h training, 25GB peak memory
```

## 6. 未来方向 (不在 V49 范围)

## 8. 元结论

**V49 路径探索完成。Scale baseline 1.2B 达到 val_ppl 2.36 (best in class),但 char-level code 数据 + diversity 阈值 0.3 限制下, 5/7 PARTIAL 是天花板。**

**V49 1.2B 是 best-in-class baseline Transformer, 接受为 V49 canonical 产物。**
